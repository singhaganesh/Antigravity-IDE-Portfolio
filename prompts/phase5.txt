─── 1. ACTIVE STATE SYNC (the "open file" illusion) ──
This is the most important part. Every IDE chrome component must react to the
current Next.js route. Use usePathname() from next/navigation in each component.

Sidebar.tsx: active file row = bg-[#37373d], border-l-2 border-white text-white
TabBar.tsx : active tab = bg-[#1e1e1e] text-white, border-t-2 color = tabBorder from map
Breadcrumb : last segment = FILE_MAP[pathname].name
StatusBar  : right side "TypeScript React" → FILE_MAP[pathname].lang

Create a shared context ActiveFileContext that wraps the root layout and exposes
the current FILE_MAP entry. All IDE chrome components consume this context.

Page transitions:
  Wrap {children} in a motion div (use framer-motion OR pure CSS):
  On route change: fade out (opacity 0, 80ms) then fade in new page (opacity 0→1, 200ms)
  This mimics switching files in VS Code
  If using pure CSS: add a className with animate-fadeUp to each page's root div

─── 4. MOBILE RESPONSIVENESS ────────────────
On screens < 768px the full sidebar cannot fit. Handle gracefully:

TitleBar: show a hamburger icon (Menu icon, lucide) on the left instead of traffic lights
ActivityBar: hidden on mobile
Sidebar:
  - Hidden by default on mobile
  - Hamburger click: slides in as full-height overlay from the left (fixed, z-50)
  - Shows the full file list; clicking a file navigates + closes the drawer
  - Outside click closes drawer
TabBar: overflow-x-auto with scrollbar-hide; show only the active tab's name on very small screens
MenuBar: hidden on mobile (< md)
Editor area: full width, padding reduces to px-5 py-8 on mobile
All section headings: scale down to text-[40px] on mobile (text-[64px] on desktop)

─── 6. PERFORMANCE ───────────────────────────
- Star particles on home page: generate inside useEffect (client only) to avoid
  hydration mismatch — initial render returns null for particles, useEffect populates them
- Wrap particle array in useMemo
- Add to globals.css:
  @media (prefers-reduced-motion: reduce) {
    *, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }
  }
- All Next.js  components should have prefetch={true} (default) for instant navigation
