1. The IDE Metaphor: File Naming & Context
Since your portfolio mimics an IDE, we need to represent this gallery as a "file." Good options would be adventures.md, biker_ganesh.config, or an image preview tab. Let's go with a Markdown file metaphor, as it allows for a mix of text and imagery.

First, update your src/context/ActiveFileContext.tsx to register the new route in the FILE_MAP:

TypeScript
// Inside src/context/ActiveFileContext.tsx
import { Camera } from 'lucide-react'; // Or an Image icon

const FILE_MAP: Record<string, FileData> = {
  // ... existing routes
  '/achievements': {
    name: 'biker_ganesh.md',
    language: 'markdown',
    icon: <Camera className="text-purple-400" size={16} />,
  },
};
2. Building the Route (src/app/achievements/page.tsx)
Create a new Next.js route for the gallery. Since you have framer-motion installed, we can make the images fade in sequentially and implement a smooth grid layout.

Here is a recommended structure for your new page:

TypeScript

3. Key Design Features for this Setup:
The "Bento Box" Grid: By using Tailwind's CSS grid (grid-cols-3 and auto-rows), you can make your most important photos (like the Sandakphu trip) take up a larger 2x2 square (col-span-2 row-span-2), while smaller updates (like gear shots or your SMK Gullwing helmet) take up standard 1x1 squares.

The IDE Hover State: When a user hovers over an image, instead of a standard generic overlay, it fades into a slightly transparent terminal background (bg-editor/80) and displays the title using code comment syntax (// International Expedition).

Sidebar Integration: Don't forget to update your src/components/ide/Sidebar.tsx to include the new path so users can actually click and open the file from the explorer pane.
